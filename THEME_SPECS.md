# Dracpurp: Refined Semantic Architecture

This document defines the core specifications for the Dracpurp aesthetic manifold, focusing on cognitive clarity and semantic grouping.

## 1. Syntax Mapping Rules

The theme groups syntax into functional categories to minimize visual noise:

| Functional Role | Semantic Group | Hex |
| :--- | :--- | :--- |
| **Logic & Structure** | `class`, `interface`, `namespace` | #B2A1E5 |
| **Data Shapes** | `type`, `struct`, `enum` | #A8B4C3 |
| **Data Vehicles** | `parameter` | #DEAC77 |
| **Executables** | `method`, `function` | #A1D3A9 |
| **Control Flow** | `keyword`, `storage` | #F272BE |
| **Workhorse** | `variable`, `property`, `string`, `number` | #CEC8B9 |

## 2. Technical Architecture

- **Semantic Primacy:** LSP semantic highlighting is enabled (`semanticHighlighting: true`) for deep contextual awareness.
- **Lineage Architecture:**
  - **Optimized:** Uses the refined muted palette for modern high-fidelity coding.
  - **Original:** Preserves the ancestral Dracula roots.
- **High Contrast:** All variants provide a High Contrast manifestation using a pure black (#000000) substrate while maintaining chromatic integrity.
- **Motion:** Italic variants apply flow exclusively to keywords, parameters, and comments.
